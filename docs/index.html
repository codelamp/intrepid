Introduction

This library is a break-away project from my theory.js

I’ve been working towards some form of object navigation for years. I’ve tried numerous approaches, only to hit up against a particular flaw or missing feature. Hopefully with Intrepid I’ve got the right balance of features and flexibility.

When dealing in object navigation there are two key factors to be aware of:

  1. The structure of the Query Language.
  2. The structure of the data being traversed.

Intrepid breaks these into two systems, the QueryInterface and the Navigator. It also makes sure that when navigating objects, you can still reference the original items. Many object navigators out there only allow you to access a copy of the data, this means making alterations to the source data is more awkward. Considering I wrote Intrepid to integrate with Mechanism.js — not being able to access the actual data refs would be useless.

What also makes this different from my previous approaches is that each targetted item of data gets wrapped with its own handler. Meaning that you can have a collection of different data items all navigated using the same Query Language.
