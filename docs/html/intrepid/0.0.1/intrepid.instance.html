chain(target) → {intrepid.instance}

Mostly used internally to return a new instance, but one that retains any information that should be kept alive between chain operations. By default, the new instance will take the current cursor, but if a new target is supplied, it will use that instead.

It should be noted that creating a new instance from a cursor will not dereference that cursor. So if providing a cursor, make sure to handle the dereferencing yourself.
