intrepid.cursorObject

The Cursor layer maintains tracking the data references so that the top level intrepid API can focus on how to navigate.

Layers

Relationships between helper entities needs to be maintained in a relative manor. There should be no global or top level references. This then allows each layer to switch out what it is using.

Each layer maintains references to the next, by way of its helpers.

+--------------------------------------------------+
| Intrepid instance                                |
| +----------------------------------------------+ |
| | CursorObject                                 | |
| | +-----------------+-----------------+--------| |
| | | targetReference | targetReference | targetR| |
| | | +-------------+ | +-------------+ | +------| |
| | | | dataHandler | | | dataHandler | | | dataH| |
| | | +-------------+ | +-------------+ | +------| |
| | +-----------------------------------+--------| |
| +----------------------------------------------+ |
+-------------------------------------------------+

This means however, that when there is a change in information. The lower entities need to update. Rather than update however, it might be best that the lower entities are just always referencing the parent.
